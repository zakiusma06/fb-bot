Build me an MVP Telegram bot in Python for product research using Meta Ads Library.

Goal:
Create a Telegram bot that guides me step by step through an extraction workflow, then runs a scraper on Meta Ads Library and saves the results into Google Sheets.

This is MVP only.
Do NOT build dashboards, product pages, ad launching, auto-learning, self-healing, or advanced admin panels.
Focus only on:
- Telegram conversational flow
- AI keyword suggestions
- Meta Ads Library extraction
- filtering
- deduplication
- Google Sheets output

Main user flow in Telegram:
1. I send /extract
2. Bot asks: how many products do you want to extract?
3. I answer with a number, for example 20
4. Bot asks for keywords
5. Bot should offer AI keyword suggestions and also allow manual keywords
6. Bot asks for countries
7. Bot asks for media type
8. Bot asks for active status
9. Bot shows a summary and asks for confirmation
10. If I confirm, bot runs the workflow
11. Bot sends progress updates if possible
12. Bot sends final summary when finished

Important extraction behavior:
The requested number means unique saved products, not raw ads scanned.
So if user asks for 20 products, the scraper should try to save 20 unique products after filtering and deduplication.
If not enough valid products are found, return partial results clearly.

Core scraper requirements:
Use Playwright in Python.
Search Meta Ads Library using the user-selected countries and keywords.
Scrape results and save product/ad data.
Add waits and retries.
Handle dynamic loading and scrolling.
Continue even if one ad fails.

Deduplication requirements:
I do not want duplicate products when multiple advertisers promote the same product.

Implement practical MVP deduplication using:
1. exact duplicate checks:
   - same ad_library_url
   - same normalized landing_page_url
   - same media_url when present
2. text similarity:
   - compare normalized product names
   - compare page titles if useful
3. image similarity:
   - if main image exists, use a lightweight image similarity approach such as perceptual hashing
   - do not overengineer this
   - use image similarity as a supporting signal, not the only signal

Deduplication decision:
- exact match => duplicate
- strong text match + strong image similarity => duplicate
- duplicate should not create a new row
- instead update duplicates_count on the existing row when possible
