FIX 3: FOR SOURCING PRICE, USE PRODUCT PAGE IMAGE, NOT FACEBOOK THUMBNAIL

Current problem:
The bot uses the Facebook Ads Library thumbnail for image search sourcing, and this sometimes gives wrong matches.

New required logic:
For sourcing price search (1688 image search), do NOT use the Facebook thumbnail as the primary image.

Image priority must be:

1. Download the main product image directly from the product page
2. If the main image fails, try other product images from the product page
3. If product page images cannot be downloaded, take a screenshot of the product from the product page
4. Only if everything above fails, optionally use the Facebook thumbnail as last fallback

Important:
The sourcing search must use the cleanest product image possible from the actual product page.

Implementation details:
- When visiting the landing page, extract the main product image URL
- Download that image locally for sourcing use
- If image download fails, try alternate product images from the page
- If all image downloads fail, take a screenshot of the visible product section from the product page
- Use that downloaded image or screenshot to perform the 1688 image search

Add logs like:

- fallback to alternate product image
- fallback to screenshot for sourcing
- skipped Facebook thumbnail because product page image available
